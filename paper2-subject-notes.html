Theory of Computation

Theory of Computation

Highest Weightage ~14 Questions Priority: HIGH

TOC is one of the highest-yield technical subjects for the HPCL officer posts examination. Focus on formal language hierarchy, grammar types, automata, pushdown automata, Turing machines, and decidability. Master the hierarchy and closure properties first because they unlock a large share of direct MCQs.

Exam Tip: ~50% of TOC questions test the Language Hierarchy and which machine accepts which language. Memorize the table below cold.
Language Hierarchy (Most Critical)
1 Regular Language (RL) → DFA / NFA / ε-NFA
2 Deterministic CFL (DCFL) → Deterministic PDA (DPDA)
3 Context-Free Language (CFL) → Non-Deterministic PDA (NPDA)
4 Context-Sensitive Language (CSL) → Linear Bounded Automata (LBA)
5 Recursive Language → Halting Turing Machine
6 Recursively Enumerable (RE) → Turing Machine (may not halt)
Subset Rule: RL ⊂ DCFL ⊂ CFL ⊂ CSL ⊂ Recursive ⊂ RE. Upper classes include all lower. If a language is Regular → it is automatically CFL, CSL, etc.
Key Concepts
Grammar

Regular Grammar Rules

Must be strictly Right-Linear or Left-Linear — never mixed. Right-Linear: S → aS or S → a. A variable can only appear at the end (right) or start (left), never in the middle.

Automata

NPDA Transition Function

δ (current_state, input_symbol, stack_top) → set of (next_state, stack_string). It takes 3 inputs: current state, current input symbol, and the current top of stack.

CFL vs DCFL

wwR vs wcwR

wwR (even palindrome, no center marker) = CFL only — DPDA cannot find the midpoint.
wcwR (with center marker c) = DCFL — marker tells the DPDA to switch.

Closure

Closure Properties — Regular

  • Union, Intersection, Complement ✓
  • Concatenation, Kleene Star ✓
  • Reversal, Homomorphism ✓
  • All boolean operations closed ✓
Closure

Closure Properties — CFL

  • Union, Concatenation, Kleene Star ✓
  • Intersection with Regular ✓
  • Intersection of two CFLs ✗
  • Complement ✗
Decidability

Decidable vs Undecidable

  • Is w ∈ L? (Regular/CFL) — Decidable
  • Is CFL empty? — Decidable
  • Halting Problem — Undecidable
  • Are two CFGs equivalent? — Undecidable
Formulas & Rules
Number of States — DFA for aⁿ where n mod k = 0 Minimum DFA states = k states (one for each remainder 0 to k-1)
Pumping Lemma for Regular Languages For any string w in L where |w| ≥ n (pumping length): w = xyz where |xy| ≤ n, |y| ≥ 1, and xyiz ∈ L for all i ≥ 0
Grammar Classification — Chomsky Hierarchy Type 0 (RE) → Type 1 (CSL, |α| ≤ |β|) → Type 2 (CFL, A → α) → Type 3 (Regular, A → aB or A → a)
Regular Expression & Automata Facts
Equivalence

RE, NFA, DFA Are Equivalent

Regular Expressions, NFA, ε-NFA, and DFA have the same expressive power. They all define exactly the set of Regular Languages.

Conversion

NFA to DFA

Subset construction may produce up to 2n DFA states from an NFA with n states. This upper-bound fact is a standard theory MCQ.

Minimization

Minimal DFA

Equivalent states are merged during minimization. The minimized DFA is unique up to renaming and has the smallest number of states for the language.

Past Year Questions — Interactive
⭐ Frequently Asked
Q1. Which of the following is NOT a Deterministic Context-Free Language (DCFL)?
A) anbn (n ≥ 1)
B) wcwR (w is a string, c is a separator)
C) wwR (even-length palindrome, no separator)
D) {anbm | n ≠ m}
Answer: C — wwR is CFL but NOT DCFL. A DPDA cannot determine the midpoint of the string non-deterministically. wcwR is DCFL because the center marker 'c' tells the DPDA when to switch from push to pop mode.
⭐ Frequently Asked
Q2. In a Non-Deterministic Pushdown Automata (NPDA), the transition function takes which three inputs?
A) Current state, next state, input symbol
B) Current state, input symbol, current top of stack
C) Input symbol, stack contents, output
D) Current state, stack contents, tape symbol
Answer: B — NPDA transition: δ(current_state, input_symbol, stack_top) → set of (next_state, stack_string). Only the top of the stack is visible, not the entire stack.
⭐ Important Concept
Q3. A grammar has productions: S → aSb | ε. The language generated is?
A) Regular — can be recognized by DFA
B) DCFL — anbn, recognized by DPDA
C) Only CFL — not DCFL
D) Context-Sensitive Language
Answer: B — This generates {anbn | n ≥ 0}, which is anbn — a classic DCFL. The grammar is Context-Free (CFG), but the language is DCFL since a DPDA can accept it deterministically.
DBMS

Database Management Systems

High Weightage ~12 Questions Priority: HIGH

DBMS questions in the HPCL officer paper usually revolve around ER diagrams, keys, integrity constraints, normalization, SQL, transactions, and relational algebra. Prioritize notation, constraints, and normal forms because they are fast marks when revised cleanly.

Exam Tip: ER diagram notation questions appear almost every year. Memorize which shape represents which concept — especially Double Line shapes and Derived/Multi-valued attributes.
ER Diagram Notation Cheat Sheet
Symbol / Notation Represents When Used
Rectangle Entity Set Any strong entity (has its own primary key)
Double Rectangle Weak Entity Entity that depends on another for existence (no own PK)
Diamond Relationship Set Connection between two entity sets
Double Diamond Identifying Relationship Links a weak entity to its owner entity
Ellipse (Oval) Attribute Property of an entity or relationship
Double Ellipse Multi-valued Attribute Attribute with multiple values (e.g., phone numbers)
Dashed Ellipse Derived Attribute Computed from another attribute (e.g., Age from DOB)
Underlined Attribute Primary Key Uniquely identifies entity in a strong entity set
Dashed Underline Partial Key Identifies weak entity (combined with owner's PK)
Integrity Constraints
Constraint

Entity Integrity

The Primary Key of a table can never be NULL. Every row must be uniquely identifiable. This is enforced at the table level.

Constraint

Referential Integrity

A Foreign Key value must either match an existing Primary Key in the referenced table, or be NULL. No dangling references allowed.

Constraint

Domain Constraint

Each attribute value must be within the defined domain of that attribute. For example, an Age column should only store positive integers.

ACID

Transaction Properties

  • Atomicity — All or nothing
  • Consistency — DB stays valid
  • Isolation — Transactions independent
  • Durability — Committed = permanent
SQL

SQL Clause Hierarchy

Mandatory: SELECT, FROM
Optional: WHERE, GROUP BY, HAVING, ORDER BY
Execution order: FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY

Joins

Types of Joins

  • INNER JOIN — matching rows only
  • LEFT JOIN — all from left + matches
  • RIGHT JOIN — all from right + matches
  • FULL JOIN — all rows, nulls for no match
Normalization Forms
Normal Form Requirement Violation Example
1NF No multi-valued / composite attributes; atomic values only Phone = {9876, 1234} in one cell
2NF 1NF + No partial dependency (non-key attribute depends on part of composite PK) Student_Name depends only on Student_ID, not (Student_ID, Course_ID)
3NF 2NF + No transitive dependency (non-key → non-key) Zip → City where Zip is not a key
BCNF For every FD X → Y, X must be a superkey Stricter than 3NF; some 3NF tables violate BCNF
Relational Algebra
Selection (σ) — Filter Rows σ_condition(R) → Returns tuples satisfying condition. All columns returned by default.
Projection (π) — Filter Columns π_A1,A2(R) → Returns only specified attributes. Duplicates are eliminated automatically.
Natural Join (⋈) R ⋈ S → Joins on common attributes automatically. Equivalent to: π_common(σ_R.A=S.A(R × S))
Keys & Functional Dependencies
Keys

Primary, Candidate, Super, Foreign

Super Key: any attribute set that uniquely identifies a tuple.
Candidate Key: minimal super key.
Primary Key: selected candidate key.
Foreign Key: references a key in another relation.

FD

Functional Dependency

X → Y means tuples equal on X must be equal on Y. Normalization questions usually revolve around whether the determinant is a key, part of a key, or a non-key attribute.

Transactions

Schedule Anomalies

Remember lost update, dirty read, and unrepeatable read. Serializable schedules preserve correctness of concurrent execution.

⭐ Frequently Asked
Q1. Which constraint states that the value of a primary key cannot be NULL?
A) Referential Integrity Constraint
B) Entity Integrity Constraint
C) Domain Constraint
D) Key Constraint
Answer: B — Entity Integrity Constraint. This rule ensures every tuple in a relation is uniquely identifiable. The Primary Key must never be null because it serves as the unique identifier of each row.
⭐ Frequently Asked
Q2. In an ER diagram, a Double Line Diamond represents which element?
A) Multi-valued Attribute
B) Weak Entity Set
C) Identifying Relationship
D) Derived Attribute
Answer: C — Identifying Relationship. A double-line (bold) diamond is used to represent the identifying relationship that connects a weak entity to its identifying (owner) entity. Remember: Double Rectangle = Weak Entity; Double Diamond = Its relationship with owner.
⭐ SQL Focus
Q3. In SQL, which of the following clauses are optional (not mandatory for a query to work)?
A) SELECT and FROM
B) FROM and WHERE
C) WHERE, GROUP BY, HAVING, ORDER BY
D) SELECT and WHERE
Answer: C. A valid SQL query only requires SELECT and FROM. SELECT * FROM employees; is a complete, valid query. WHERE, GROUP BY, HAVING, and ORDER BY are all optional clauses used for filtering, grouping, and sorting.
Operating Systems

Operating Systems

High Weightage ~11 Questions Priority: HIGH

Operating Systems for the HPCL officer exam centers on process management, memory management, scheduling, file systems, deadlocks, and synchronization. This section is often direct and scoring if you memorize state transitions, scheduler roles, and timing formulas.

Exam Tip: Remember: New and Terminated states reside in Secondary Memory. Ready, Running, and Waiting states are in Primary Memory (RAM). This is a direct exam question.
Process State Lifecycle
Process State → Memory Location
New → Secondary Memory Ready → Primary (RAM) Running → Primary (RAM) Waiting → Primary (RAM) Terminated → Secondary Memory
OS Schedulers
Scheduler Alias Moves From Moves To Frequency
Long-Term Scheduler Job Scheduler New (Secondary) Ready (RAM) Slow (controls degree of multiprogramming)
Short-Term Scheduler CPU Scheduler Ready (RAM) Running (CPU) Very Fast (runs every few milliseconds)
Mid-Term Scheduler Swapper Waiting/Running Suspended/Ready Medium (reduces multiprogramming when needed)
Key Concepts
Memory

Paging — Address Translation

Logical address = Page Number + Offset
If logical space = 2n bytes and page size = 2m bytes:
• Offset bits = m
• Page number bits = n - m

Kernel Access

System Calls

When a user program needs OS kernel services, it uses a System Call via an Application Program Interface (API). Examples: read(), write(), fork(), exec().

File System

File Allocation Table (FAT)

Maintains one entry per disk block, indexed by block number. Used in FAT12, FAT16, FAT32 file systems. Each entry points to the next block or indicates end-of-file.

Deadlock

Deadlock Conditions (All 4 Required)

  • Mutual Exclusion
  • Hold and Wait
  • No Preemption
  • Circular Wait
Scheduling

CPU Scheduling Algorithms

  • FCFS — First Come First Served (non-preemptive)
  • SJF — Shortest Job First
  • SRTF — Shortest Remaining Time First
  • Round Robin — time quantum based
  • Priority — highest priority runs first
Synchronization

Critical Section Problem

Three requirements: Mutual Exclusion (only one process at a time), Progress (if no one in CS, a waiting process should enter), Bounded Waiting (no starvation).

Formulas
Page Addressing Example Logical space = 4096 = 2^12, Page size = 512 = 2^9
→ Offset bits = 9, Page number bits = 12 - 9 = 3 bits
Process Size in Paging Process Size = Number of Pages × Page Size
Turnaround Time TAT = Completion Time - Arrival Time
Waiting Time WT = Turnaround Time - Burst Time
Memory Management & Synchronization
Fragmentation

Paging vs Segmentation

Paging removes external fragmentation but may cause internal fragmentation. Segmentation can suffer from external fragmentation because segment sizes are variable.

Semaphore

Binary vs Counting Semaphore

Binary semaphore behaves like a lock with values 0/1. Counting semaphore manages multiple instances of a resource using an integer value.

Replacement

Page Replacement Policies

FIFO, LRU, and Optimal are the most important policies. FIFO can show Belady's anomaly; Optimal gives the theoretical minimum page faults.

⭐ Direct Question
Q1. In an operating system, the "Ready Queue" is maintained in which memory?
A) Secondary Memory (Hard Disk)
B) Primary Memory (RAM)
C) Cache Memory
D) Registers
Answer: B — Primary Memory (RAM). The Ready, Running, and Waiting queues are all in RAM. Only the New and Terminated process states reside in Secondary Memory (disk).
⭐ Calculation
Q2. If logical address space = 4096 units and page size = 512 units, how many bits are needed for the page number?
A) 9 bits
B) 12 bits
C) 3 bits
D) 4 bits
Answer: C — 3 bits. Total bits = log₂(4096) = 12 bits. Offset bits = log₂(512) = 9 bits. Page number bits = 12 - 9 = 3 bits (which gives 2³ = 8 page frames).
⭐ Concept
Q3. When a user program needs to access the OS kernel for services, it uses a ___?
A) Interrupt Request
B) Device Driver
C) System Call
D) Semaphore
Answer: C — System Call. A system call is the programmatic way user-space programs request services from the OS kernel. It acts through an API. Examples: fork(), open(), read(), exec().
COA

Computer Organization & Architecture

Medium Weightage ~10 Questions Priority: MEDIUM

COA tests CPU organization, memory hierarchy, addressing modes, instruction formats, pipelining, number systems, and digital circuits. For the officer-post exam, keep the focus on comparisons, conversions, and simple machine-level calculations.

Memory Hierarchy (Fastest → Slowest)
Level Type Location Speed Cost/Size
1 Registers Inside CPU Fastest Tiny (bytes), Expensive
2 Cache (L1/L2/L3) Between CPU & RAM Very Fast KB–MB, Expensive
3 Main Memory (RAM) On Motherboard Fast GB, Moderate
4 Secondary Storage Hard Disk / SSD Slow TB, Cheap
5 Tertiary Storage Tape / Optical Slowest Very Large, Cheapest
Addressing Modes
Mode

Immediate Addressing

Operand value is directly embedded in the instruction itself. No memory access needed. Fastest. Example: MOV R1, #5 (load value 5 into R1).

Mode

Register Addressing

Instruction contains the name/address of a register. The actual operand is stored inside the register. Very fast — no memory access.

Mode

Direct (Absolute) Addressing

The instruction contains the memory address of the operand. One memory access required. Example: LOAD 2000 fetches data from address 2000.

Mode

Indirect Addressing

Instruction has an address that points to another address which holds the operand. Two memory accesses needed. Useful for pointers.

Mode

Indexed Addressing

Effective Address = Base Address + Index Register. Used for accessing arrays. The index register holds the displacement.

Pipelining

Pipeline Throughput

Once the pipeline is fully filled, one instruction completes per clock cycle. Pipelining improves throughput, not individual instruction latency. Hazards: Structural, Data, Control.

Formulas & Calculations
Number Base Conversion (Any Base → Decimal) N(base X) = d₀×X⁰ + d₁×X¹ + d₂×X² + ...
E.g., 1011(2) = 1×8 + 0×4 + 1×2 + 1×1 = 11
Maximum Links in Mesh Topology For n nodes: Max links = n(n−1) / 2
Combinational Circuit Output Bits For 2 inputs of k bits each: max sum = (2k−1) + (2k−1) = 2(k+1)−2
Output bits needed = k + 1 bits
E.g., 3-bit + 3-bit: max = 7+7 = 14 → needs 4 bits
CPU & Number System Essentials
Instruction Cycle

Fetch → Decode → Execute

The CPU fetches an instruction, decodes the opcode, and then executes the operation. Questions often ask the correct order or the role of the Program Counter and Instruction Register.

Signed Binary

1's and 2's Complement

1's complement flips all bits. 2's complement is 1's complement plus 1, and it is the standard representation used in signed binary arithmetic.

Architecture

RISC vs CISC

RISC favors simple instructions and many registers. CISC supports more complex instructions and addressing modes. This comparison is a frequent direct concept question.

Past Year Questions — Interactive
⭐ Frequently Asked
Q1. Which statement about Cache Memory is CORRECT?
A) It is slower than RAM but faster than disk
B) It is inside the Secondary Storage
C) It is a fast, small memory placed between the CPU and Main Memory
D) It permanently stores the operating system
Answer: C. Cache is a small, high-speed memory layer placed between the CPU and RAM. Its purpose is to store frequently accessed data so the CPU doesn't have to wait for slower RAM. L1 is closest to CPU and fastest.
⭐ Calculation Question
Q2. A combinational circuit takes two 3-bit numbers as inputs and outputs their sum. How many output lines are needed?
A) 3
B) 4
C) 6
D) 5
Answer: B — 4 output lines. Maximum 3-bit number = 7 (111₂). Maximum sum = 7 + 7 = 14 (1110₂). This requires 4 bits to represent. So the circuit needs 4 output lines.
⭐ Pipeline
Q3. Which is TRUE about a k-stage pipeline processor?
A) Each instruction takes 1 clock cycle to execute
B) k instructions can run simultaneously, none sharing stages
C) After the pipeline is fully filled, one instruction completes per clock cycle
D) The first instruction completes in 1 clock cycle
Answer: C. Initially, it takes k cycles to fill the pipeline (fill latency = k cycles for first instruction). Once filled, one instruction completes per cycle — this is ideal throughput. First instruction still takes k cycles.

Algorithms & Data Structures

Medium Weightage ~8-9 Questions Priority: MEDIUM

Algorithms questions usually test graph traversals, tree properties, sorting, greedy vs dynamic programming, and Big-O complexity analysis. In HPCL officer exams, identification questions are common, so learn which paradigm fits which problem.

Master Theorem
Recurrence Relation
T(n) = aT(n/b) + Θ(nk logp n)

Compare a with b^k:
Case 1: a > b^k → T(n) = Θ(nlogba)
Case 2: a = b^k
    If p > -1 → T(n) = Θ(nk logp+1 n)
    If p = -1 → T(n) = Θ(nk log log n)
Case 3: a < b^k
    If p ≥ 0 → T(n) = Θ(nk logp n)

Tree Traversals — BST Properties
BST Property

In-Order Traversal

In-Order (Left → Root → Right) of a Binary Search Tree always produces nodes in non-decreasing (sorted ascending) order. This is the most tested BST property.

Traversal

Pre-Order & Post-Order

Pre-Order (Root → Left → Right): Used to copy a tree.
Post-Order (Left → Right → Root): Used to delete/evaluate expression trees.

Graph

BFS vs DFS

BFS — Uses Queue. Explores level by level. Finds shortest path in unweighted graphs.
DFS — Uses Stack (or recursion). Explores depth-first. Used for cycle detection, topological sort.

Greedy vs DP

Key Distinction

Greedy: Makes locally optimal choice at each step, never reconsiders. Works for Fractional Knapsack.
DP: Solves overlapping subproblems optimally. Required for 0/1 Knapsack.

Sorting

Sorting Complexity

  • Bubble/Selection/Insertion — O(n²)
  • Merge Sort / Heap Sort — O(n log n)
  • Quick Sort — O(n log n) avg, O(n²) worst
  • Counting/Radix Sort — O(n+k)
MST

Minimum Spanning Tree

MST has exactly V-1 edges for V vertices. Kruskal's builds by adding minimum weight edges without cycles. Prim's grows from a starting vertex by adding cheapest adjacent edge.

Formulas
Matrix Chain Multiplication — DP Time Complexity T(n) = Θ(n³) when solved using Dynamic Programming
Height of Complete Binary Tree Height h = ⌊log₂ n⌋ where n = number of nodes
MST Edge Count A Minimum Spanning Tree of a graph with V vertices has exactly V - 1 edges
Search, Hashing & Core Analysis
Searching

Binary Search Requirement

Binary Search requires a sorted array or search space. Its time complexity is O(log n) because each step halves the remaining search interval.

Hashing

Collision Resolution

Important techniques are chaining, linear probing, quadratic probing, and double hashing. Primary clustering is strongly associated with linear probing.

Asymptotic

O, Ω, Θ

O gives an upper bound, Ω gives a lower bound, and Θ gives a tight bound. These notations are regularly tested in one-line theory questions.

Past Year Questions — Interactive
⭐ Direct PYQ
Q1. Which of the following problems CANNOT be solved by a Greedy algorithm?
A) Fractional Knapsack
B) Dijkstra's Shortest Path
C) 0/1 Knapsack
D) Prim's MST
Answer: C — 0/1 Knapsack. The 0/1 Knapsack requires Dynamic Programming because each item is either fully taken or left. Greedy fails here because locally optimal choices don't lead to globally optimal solutions. Fractional Knapsack IS solvable by Greedy (take fractional amounts).
⭐ Direct PYQ
Q2. Which Greedy algorithm uses the Disjoint Set data structure?
A) Dijkstra's Shortest Path
B) Kruskal's Minimum Spanning Tree
C) Prim's MST
D) Fractional Knapsack
Answer: B — Kruskal's MST Algorithm. Kruskal's adds edges in increasing order of weight and uses the Disjoint Set (Union-Find) structure to efficiently detect and avoid cycles. If adding an edge creates a cycle (both vertices in same set), it is skipped.
⭐ Direct PYQ
Q3. Which traversal of a BST gives nodes in non-decreasing (sorted ascending) order?
A) Pre-Order Traversal
B) In-Order Traversal
C) Post-Order Traversal
D) Level-Order Traversal
Answer: B — In-Order Traversal (Left → Root → Right). For a Binary Search Tree, an In-Order traversal visits nodes in ascending sorted order. This is a fundamental BST property. Pre-Order gives a copy-friendly sequence; Post-Order is used in expression tree evaluation.

Routing Algorithm Comparison
Distance Vector

RIP — Routing Information Protocol

Uses Bellman-Ford algorithm. Shares full routing table with neighbors. Prone to count-to-infinity problem. Max hop count = 15.

Link State

OSPF — Open Shortest Path First

Uses Dijkstra's algorithm. Each router holds a full topology map. Faster convergence, more scalable. Sends link-state advertisements (LSAs).

Path Vector

BGP — Border Gateway Protocol

Used for inter-AS (internet-level) routing. Shares full path to avoid loops. Policy-based, highly scalable but slow convergence.

Quick Comparison

RIP vs OSPF

Algorithm Bellman-Ford Dijkstra

Convergence Slow Fast
Max hops 15 Unlimited
Network Devices & Domain Facts
Layer 1

Hub

Works at the Physical Layer. It broadcasts incoming bits to all ports and does not filter traffic intelligently.

Layer 2

Switch / Bridge

Works at the Data Link Layer using MAC addresses. Each switch port creates a separate collision domain, improving LAN performance.

Layer 3

Router

Works at the Network Layer using IP addresses. Routers separate broadcast domains and forward packets between networks.

Key Formulas & Delay Metrics
Transmission Delay
Tt = L / B
L = Packet length (bits), B = Bandwidth (bps)
Example: 1000-bit packet over 1 Mbps → Tt = 1 ms
Propagation Delay
Tp = d / v
d = Distance (m), v = Signal speed (~2×108 m/s in copper)
Example: 200 km → Tp = 200,000 / 2×108 = 1 ms
Round Trip Time
RTT ≈ 2 × Tp
Assumes processing and queuing delays are negligible
Note: Used to calculate TCP timeout and window size
Stop & Wait Efficiency
η = Tt / (Tt + 2Tp)
Efficiency of stop-and-wait ARQ protocol
Tip: Higher propagation → lower efficiency
Usable Hosts / Subnet
2(32−n) − 2
n = CIDR prefix length. Subtract network + broadcast addresses.
Example: /26 → 26 − 2 = 62 hosts
Bandwidth-Delay Product
BDP = B × Tp
Maximum data "in flight" in the network at once (bits)
Example: 1 Mbps × 10 ms = 10,000 bits in transit
Past Year Questions — Interactive
Score 0 / 0
⭐ Direct PYQ
Q1. Which layer of the OSI model is responsible for routing?
A) Data Link Layer
B) Network Layer
C) Transport Layer
D) Physical Layer
Answer: B — Network Layer. Layer 3 handles logical IP addressing and routes packets across multiple networks using routers. The Data Link Layer (L2) handles MAC-level delivery only within a single segment.
🔥 Numerical
Q2. What is the maximum number of usable hosts per subnet for a /26 prefix?
A) 126
B) 62
C) 64
D) 254
Answer: B — 62. /26 → host bits = 32−26 = 6. Total addresses = 26 = 64. Subtract 1 (network) + 1 (broadcast) = 62 usable hosts.
🔥 Frequent Concept
Q3. Which routing protocol uses the Bellman-Ford algorithm and is prone to count-to-infinity?
A) OSPF
B) BGP
C) RIP
D) EIGRP
Answer: C — RIP. RIP uses Bellman-Ford and shares routing tables with neighbors. The count-to-infinity problem occurs when a link fails and routers keep incrementing the hop count. Max hops = 15; 16 = infinity (unreachable).
⭐ Direct PYQ
Q4. TCP uses a ___-way handshake to establish a connection.
A) 2-way
B) 3-way
C) 4-way
D) 5-way
Answer: B — 3-way handshake. TCP establishes a connection with SYN → SYN-ACK → ACK. Note: TCP uses a separate 4-way handshake to close a connection (FIN → ACK → FIN → ACK).
🔥 Numerical
Q5. A packet of 5000 bits is sent over a 1 Mbps link. Propagation delay = 10 ms. Total end-to-end delay?
A) 15 ms
B) 10 ms
C) 5 ms
D) 20 ms
Answer: A — 15 ms. Tt = L/B = 5000 / 106 = 5 ms. Tp = 10 ms. Total = 5 + 10 = 15 ms.
Software Engineering

Software Engineering

Low Weightage ~6 Questions Priority: LOW

Software Engineering is usually lighter in weightage, but it remains a useful scoring area for the HPCL officer exam if revised smartly. Focus on process models, testing, cohesion/coupling, maintenance, SRS, and COCOMO-style comparison questions.

Important: Software Engineering questions are usually definition-based and comparison-heavy. Process models, testing levels, cohesion/coupling, maintenance, and SRS are the most dependable scoring concepts.
1. Software Process Models (Core Understanding)

Waterfall Model

Sequential flow: Requirements → Design → Implementation → Testing → Deployment → Maintenance.

Key Idea: No phase overlap.
Advantages: Simple, easy to manage.
Limitations: No flexibility, late error detection.

Exam Insight: Not suitable when requirements are changing.

Agile Model

Iterative & incremental development with continuous feedback.

Key Idea: Deliver working software in small cycles (sprints).
Advantages: Flexible, customer-centric.
Limitation: Hard to predict cost/time precisely.

Exam Insight: Best when requirements evolve.

Spiral Model

Combines iterative development with risk analysis.

Each loop = Planning → Risk Analysis → Engineering → Evaluation.
Radius represents cost, angle represents progress.

Exam Insight: Used in high-risk projects.

2. Software Testing (Hierarchy + Concepts)

Levels of Testing

Unit Testing: Smallest module, White-box testing.
Integration Testing: Checks interaction between modules.
System Testing: Entire system, Black-box testing.
Acceptance Testing: User validation (Alpha: internal, Beta: external).

White-box vs Black-box

White-box: Internal logic, code coverage, path testing.
Black-box: Input-output behavior, no code knowledge.

Exam Trap: Unit = White-box, System = Black-box (usually asked).

3. Design Principles (Very Important)

Cohesion (Within Module)

Degree to which elements inside a module belong together.

Types (Worst → Best): Coincidental → Logical → Temporal → Procedural → Communicational → Sequential → Functional.

Goal: HIGH cohesion.

Coupling (Between Modules)

Degree of interdependence between modules.

Types (Worst → Best): Content → Common → Control → Stamp → Data.

Goal: LOW coupling.

Design Rule

Best design = High Cohesion + Low Coupling

Why?
- Easier maintenance
- Better scalability
- Less bug propagation

4. Software Maintenance

Types of Maintenance

Corrective: Fix defects.
Adaptive: Adapt to environment changes.
Perfective: Add/improve features (MOST effort).
Preventive: Improve maintainability.

5. SRS (Software Requirements Specification)

Properties of Good SRS

- Correct
- Complete
- Consistent
- Unambiguous
- Verifiable

Role: Acts as a contract between client and developer.

6. Effort Estimation (COCOMO)

COCOMO Model

Effort = a × (KLOC)b

Types:
- Organic (small projects)
- Semi-detached
- Embedded (complex systems)

Used for effort, time, and cost estimation.

7. Verification, Validation & Reliability

Verification vs Validation

Verification: Are we building the product right?
Validation: Are we building the right product?

Verification is process-oriented; validation is user-need oriented.

Error, Fault, Failure

Error: human mistake.
Fault: defect in software.
Failure: incorrect external behavior during execution.

Software Reliability

Reliability is the probability that software performs without failure for a specified time under stated conditions. It is different from correctness and maintainability.

8. Important Formula
Cyclomatic Complexity
\( V(G) = E - N + 2P \)
Measures number of independent paths → defines minimum test cases needed.
PYQ — Conceptual
Which combination is best for software design?
A) High coupling, Low cohesion
B) Low coupling, Low cohesion
C) Low coupling, High cohesion
D) High coupling, High cohesion
Answer: C
This combination ensures modular, maintainable, and scalable systems.
Programming

Programming & Data Structures

High Weightage ~10 Questions Priority: HIGH

Programming & Data Structures is a high-return area because many HPCL officer questions are direct output, memory, OOP, pointers, and code-trace questions. Strong fundamentals here can convert this section into some of the quickest marks in Paper 2.

Important: Programming questions in HPCL Paper 2 are usually built around OOP basics, memory behavior, pointer interpretation, parameter passing, and short output-tracing code snippets.
1. Object-Oriented Programming (Core Concepts)

Encapsulation

Binding data + methods together.
Achieved using private, protected, public.

Goal: Data hiding.

Abstraction

Hiding internal complexity, showing only essentials.
Achieved using abstract classes / interfaces.

Example: Driving a car without knowing engine details.

Inheritance

One class acquires properties of another.
Promotes code reuse.

Types: Single, Multiple, Multilevel, Hierarchical.

Polymorphism

One interface, multiple implementations.

Compile-time: Function overloading
Run-time: Function overriding

2. Important Language Concepts

Java Keywords

this → current object
super → parent class
static → class-level (shared memory)
final → constant / no override

Static Variables

Initialized only once.
Stored in static memory area.
Retains value across function calls.

Exam Trap: Not reinitialized each call.

3. Memory Model (Very Important)

Stack Memory

Stores function calls + local variables.
Fast allocation.
Automatically managed.

Exam Insight: Function recursion uses stack frames.

Heap Memory

Dynamic memory allocation (malloc, new).
Slower but flexible.
Requires manual deallocation.

Exam Trap: Memory leaks if not freed.

4. Pointers & Arrays (Critical Area)

Pointer Basics

Stores address of a variable.
*ptr → value at address
&x → address of x

Pointer Arithmetic

ptr + 1 → moves by size of data type.
Example: int pointer → +4 bytes.

Exam Trap: Not always +1 byte.

Arrays & Pointers

Array name acts as pointer to first element.
arr[i] = *(arr + i)

Important: Arrays ≠ pointers (but closely related).

5. Parameter Passing

Pass by Value

Copy of variable is passed.
Changes do NOT affect original variable.

Pass by Reference

Address is passed.
Changes reflect in original variable.

C: via pointers
C++: reference variables

6. Code Execution Patterns (Exam Critical)

Static Variable Behavior

Value persists across function calls.
Initialized only once.

Loop + Condition Traps

Off-by-one errors (i < n vs i <= n).
Infinite loops due to missing updates.

Pointer + Function

Modifying via pointer changes original value.
Common in swap problems.

7. Core Data Structures Facts

Stack vs Queue

Stack: LIFO, used in recursion and function calls.
Queue: FIFO, used in scheduling and BFS.

Push/Pop belong to stack, Enqueue/Dequeue belong to queue.

Linked List Basics

Each node stores data + link.
Dynamic size, non-contiguous memory allocation.

Insertion/deletion is easy after locating a node, but random access is slow.

Tree and Graph Facts

A tree with n nodes has n-1 edges.
Binary tree node degree is at most 2.

Queue and stack concepts frequently connect with traversal questions.
